# Лабораторна робота №3 - REST API з SQLite

## Варіант №11: Сервіс чергувань у лабораторії
Бекенд-система для керування графіком чергувань студентів, що дозволяє відстежувати зміни та обробляти заявки на заміну чергових.

## Як запустити проєкт
1. **Встановіть залежності:**
   ```bash
   npm install
2. **Запустіть сервер у режимі розробки:**
   ```bash
   npm run dev
3. **Порт за замовчуванням: Сервер запуститься на http://localhost:3000**

## База даних та ініціалізація
**СУБД:** SQLite.

**Файл бази:** Автоматично створюється за шляхом data/app.db.

**Схема:** Таблиці створюються автоматично при першому старті за допомогою SQL-команд CREATE TABLE IF NOT EXISTS.

**Seed (Тестові дані):** Реалізовано механізм автоматичного наповнення бази. Якщо таблиця користувачів порожня, система автоматично додає 6 тестових записів (по 2 для кожної сутності).

## Схема БД (Структура)
### В системі реалізовано 3 взаємопов'язані сутності:

1. **Users (Користувачі):** id, email (unique), name, createdAt.

2. **Shifts (Чергування):** id, userId, date, status, createdAt.
    ```bash
    Зв'язок: FOREIGN KEY (userId) REFERENCES Users(id) ON DELETE CASCADE.

3. **SwapRequests (Заявки на заміну):** id, shiftId, reason, status, createdAt.
    ```bash
    Зв'язок: FOREIGN KEY (shiftId) REFERENCES Shifts(id) ON DELETE CASCADE.

## Приклади запитів (cURL)
**Отримати всі заявки на заміну (3-тя сутність)**
    
    curl http://localhost:3000/api/swap-requests
**Складний запит (Фільтрація + Сортування + Ліміт)**

Демонстрація виконання вимоги на використання блоків WHERE, ORDER BY та LIMIT в одному запиті:
   
    curl "http://localhost:3000/api/shifts?userId=1&sort=date&order=desc&limit=5"
**Створити нове чергування**

    curl -X POST http://localhost:3000/api/shifts -H "Content-Type: application/json" -d '{"userId": 1, "date": "2026-05-10", "status": "Planned"}'
**Отримати користувача за ID**

    curl http://localhost:3000/api/users/1

## Технічні особливості
**Повний CRUD:** Реалізовано методи POST, GET (list/id), PUT та DELETE для основних сутностей.

**Централізована обробка помилок:** Middleware формує уніфіковані JSON-відповіді для помилок 400, 404 та 500.

**Логування:** Сервер логує кожен HTTP-запит (метод, шлях, статус-код) та ключові події бази даних (ініціалізація схеми, додавання seed-даних).

# Лабораторна робота №4 - Інтеграція клієнтської та серверної частин 

## Опис проєкту
Веб-додаток для керування розкладом чергувань студентів у лабораторії. Проєкт складається з двох частин:
1. **Frontend (Клієнт):** HTML, CSS, Vanilla JavaScript.
2. **Backend (Сервер):** Node.js, Express 5, база даних SQLite.

## Що реалізовано (вимоги на «добре»)
Проєкт виконано згідно з критеріями на оцінку **«добре»**:
* **Реалізація CRUD:** Фронтенд успішно виконує створення, читання, редагування та видалення записів через REST API бекенду.
* **Архітектура клієнта:** Мережевий шар (HTTP-запити через `fetch`) винесено в окремий модуль `apiClient.js` і повністю відокремлено від логіки інтерфейсу.
* **Обробка станів:** Інтерфейс коректно реагує на 4 базові стани: `loading` (завантаження), `empty` (порожня БД), `success` (дані отримано) та `error` (централізована обробка помилок).
* **Узгодження контрактів (DTO):** Фронтенд адаптовано під структуру бази даних сервера (додано фіксований `userId`, налаштовано безпечну обробку відсутніх полів та локалізацію статусів).
* **Налаштування CORS:** На бекенді підключено middleware `cors` із суворим білим списком (Whitelist) для дозволених локальних адрес клієнта (`http://127.0.0.1:5500`, `http://localhost:5500`).

## 🛠 Як запустити проєкт

### 1. Запуск сервера (Backend)
1. Відкрийте термінал у папці `backend`.
2. Встановіть залежності: `npm install`
3. Запустіть сервер: `npm run dev` (або `npm start`)
4. Сервер запуститься на `http://localhost:3000` (БД `app.db` створиться автоматично).

### 2. Запуск клієнта (Frontend)
1. Відкрийте папку `frontend` у редакторі (наприклад, VS Code).
2. Запустіть файл `index.html` за допомогою розширення **Live Server**.
3. Додаток буде доступний за адресою `http://127.0.0.1:5500/index.html`.

## Структура Frontend-частини
* `src/config.js` — конфігурація (адреса бекенду).
* `src/apiClient.js` — мережевий шар (взаємодія з API).
* `src/ui.js` — візуальний шар (DOM-маніпуляції, рендер таблиці, повідомлення).
* `src/main.js` — головний скрипт (зв'язування подій та логіки).
